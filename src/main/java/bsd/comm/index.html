The bsd.comm Driver for javax.comm

The core Java APIs do not allow you to read/write serial/parallel/etc ports, because only on UNIX and do such things have entries in the filesystem namespace. The UNIX "device special file" is a great idea, and it's too bad MacOS and that thing from Bill Gates never adopted it. But they didn't, and now it's too late.

To the rescue comes The Java Communications API. However, the javax.comm package, as it's known, does not provide direct support for platforms other than Solaris and Win32. As you might expect, the code for dealing with badu rates, parity settings, etc., is rather platform-specific. However, there is no great difficulty in controlling the terminal on BSD-derived systems, and writing a bit of "native code" to handle this is pretty easy. I've made a start on it, and here it is. This source is for the OpenBSD flavor, but should compile without much change on the other BSD platforms.
