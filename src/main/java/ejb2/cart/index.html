Cart Bean?

This is not a fully-working cart bean, just a few scratchings. Should a Cart be implemented as:

  • A servlet? Light weight, not persistent
  • A stateful session bean? Moderate weight, maybe more scaleable
  • An entity bean? Heavy weight, but persistent!
The point is, you have to decide what is the best way to implement your cart, based on such factors as the cost of items you sell, the volume of your site, and so on.

At any rate, here are some starting points for the stateful session version:

  • CartHome.java, the home interface
  • Cart.java, the remote interface
  • CartBean.java, the bean implementation
